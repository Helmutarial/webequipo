# Patrones de interfaz

## Home

Usar una composición de alto impacto con el próximo partido o último resultado como bloque principal. Acompañar con noticias y estadísticas resumidas.

## Partido

Mostrar competición, fecha, local/visitante, rival, marcador, estado y MVP. En móvil, apilar la información sin reducir demasiado el tamaño del marcador.

## Plantilla

Usar tarjetas de jugador con fotografía, dorsal, nombre y posición. Los datos sensibles deben quedar fuera de la zona pública.

## Estados

Diseñar siempre loading, vacío, error y contenido real. El estado vacío debe explicar qué falta y, en el panel privado, ofrecer una acción clara.
